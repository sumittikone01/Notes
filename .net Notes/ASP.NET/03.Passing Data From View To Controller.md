## 1. The Concept — Why & How

When a user fills a **form in the View** and clicks submit, that data travels from the browser to the server (Controller). The Controller needs to **receive and read** this data.

```
┌────────────────────────────────────────────┐
│              VIEW (.cshtml)                │
│                                            │
│   <form action="/Home/ShowAddedPerson">    │
│       <input name="id"       value="12" /> │
│       <input name="name"     value="Tom"/> │
│       <input name="mobileNo" value="999"/> │
│       <input name="address"  value="NYC"/> │
│       <button type="submit">Submit</button>│
│   </form>                                  │
└─────────────────────┬──────────────────────┘
                      │   HTTP Request (form data submitted)
                      ▼
┌────────────────────────────────────────────┐
│             CONTROLLER                     │
│                                            │
│   How do I READ the form data here?        │
│                                            │
│   4 ways:                                  │
│   1. Action Parameters   (simplest)        │
│   2. Request.Form        (manual reading)  │
│   3. IFormCollection     (collection obj)  │
│   4. Model Binding       (best/cleanest)   │
└────────────────────────────────────────────┘
```

> **Key Rule:** The `name` attribute of each `<input>` in the form is what gets sent to the server. Whatever you name your inputs — that's how the Controller identifies them.

## 2. Method 1 — Action Method Parameters

> **Simplest method.** The controller action declares parameters whose **names match the form input names exactly**. ASP.NET automatically maps the values.

### How It Works

```
View:  <input name="id" />  <input name="name" />  <input name="mobileNo" />
                  │                   │                       │
                  └───────────────────┴───────────────────────┘
                              names must match ↓
Controller:  public IActionResult ShowAddedPerson(int id, string name, long mobileNo, string address)
```

### View — AddPerson.cshtml

```html
<h1>Add Person Form</h1>

<!-- GET form — data goes in URL as query string -->
<form name="fr" action="/Home/ShowAddedPerson">
    <div>
        <label>Id</label>
        <input type="number" name="id" />
    </div>
    <div>
        <label>Name</label>
        <input type="text" name="name" />
    </div>
    <div>
        <label>Mobile No</label>
        <input type="number" name="mobileNo" />
    </div>
    <div>
        <label>Address</label>
        <input type="text" name="address" />
    </div>
    <button type="submit">Add Person</button>
</form>
```

### Controller — HomeController.cs

```csharp
// Parameter names MUST match the input name="" attributes exactly
public IActionResult ShowAddedPerson(int id, string name, long mobileNo, string address)
{
    // ASP.NET automatically fills these parameters from the form/URL data
    Person person = new Person()
    {
        Id       = id,
        Name     = name,
        MobileNo = mobileNo,
        Address  = address
    };

    return View(person);   // Pass the built object to View
}
```

### Result View — ShowAddedPerson.cshtml

```html
@model TestViewTypes.Models.Person

<div>
    <h2>Data Received from the View</h2>
    <p>Id:       @Model.Id</p>
    <p>Name:     @Model.Name</p>
    <p>MobileNo: @Model.MobileNo</p>
    <p>Address:  @Model.Address</p>
</div>
```

### What Happens Step by Step

```
1. User fills form and clicks submit
2. URL becomes: /Home/ShowAddedPerson?id=12&name=Tom&mobileNo=999&address=NYC
3. ASP.NET sees the action method signature:
       ShowAddedPerson(int id, string name, long mobileNo, string address)
4. It matches:  id=12, name="Tom", mobileNo=999, address="NYC"
5. Passes them automatically into the method
```

### Key Points

```
✔ Simplest to write and understand
✔ ASP.NET does the mapping automatically by matching names
✔ Works with GET requests (data in URL)
✔ Parameters can be any type: int, string, long, bool, etc.
✗ If names don't match exactly → parameter will be null or 0
✗ Gets messy with many fields (too many parameters)
```

## 3. Method 2 — Request Object

> **Manual reading.** You use `Request.Form["fieldName"]` inside the controller to manually read each form field by name. The form must use `method="post"`.

### How It Works

```
View:  <form method="post">  <input name="id" />  <input name="name" />
                                      │                     │
              Controller reads:       ▼                     ▼
                              Request.Form["id"]    Request.Form["name"]
```

### View — AddPersonR.cshtml

```html
<h1>Add Person Form (Request Object)</h1>

<!-- method="post" is REQUIRED for Request.Form to work -->
<form name="fr" action="/Home/ShowsAddedPerson" method="post">
    <div>
        <label>Id</label>
        <input type="number" name="id" />
    </div>
    <div>
        <label>Name</label>
        <input type="text" name="name" />
    </div>
    <div>
        <label>Mobile No</label>
        <input type="number" name="mobileNo" />
    </div>
    <div>
        <label>Address</label>
        <input type="text" name="address" />
    </div>
    <button type="submit">Add Person</button>
</form>
```

### Controller — HomeController.cs

```csharp
[HttpPost]   // Must match POST method on form
public IActionResult ShowsAddedPerson()
{
    // Manually read each field from the posted form data
    ViewBag.Id       = Request.Form["id"];
    ViewBag.Name     = Request.Form["name"];
    ViewBag.MobileNo = Request.Form["mobileNo"];
    ViewBag.Address  = Request.Form["address"];

    return View();
}
```

### Result View — ShowsAddedPerson.cshtml

```html
<div>
    <h2>Data Received from the View</h2>
    <p>Id:       @ViewBag.Id</p>
    <p>Name:     @ViewBag.Name</p>
    <p>MobileNo: @ViewBag.MobileNo</p>
    <p>Address:  @ViewBag.Address</p>
</div>
```

### Key Points

```
✔ Simple to understand — you explicitly read each field
✔ Works when you need fine-grained control over what you read
✔ Good for reading just one or two specific fields
✗ Must use [HttpPost] and form method="post"
✗ All values come as strings — you must convert types manually
✗ Very repetitive with many fields
✗ Messy — not recommended for real projects
```

## 4. Method 3 — FormCollection (IFormCollection)

> **Collection-based reading.** Instead of reading `Request.Form` directly, you accept an `IFormCollection` parameter. ASP.NET automatically injects the entire form's data into it. You then read from it like a dictionary.

### How It Works

```
View:  <form method="post">  all inputs  </form>
                    │
                    │  ASP.NET collects ALL form fields
                    ▼
Controller:  ShowIFormAddedPerson(IFormCollection form)
                    │
             form["id"]        → reads id
             form["name"]      → reads name
             form["mobileNo"]  → reads mobileNo
```

### View — AddIFormPerson.cshtml

```html
<h1>Add Person Form (IFormCollection)</h1>

<form name="fr" action="/Home/ShowIFormAddedPerson" method="post">
    <div>
        <label>Id</label>
        <input type="number" name="id" />
    </div>
    <div>
        <label>Name</label>
        <input type="text" name="name" />
    </div>
    <div>
        <label>Mobile No</label>
        <input type="number" name="mobileNo" />
    </div>
    <div>
        <label>Address</label>
        <input type="text" name="address" />
    </div>
    <button type="submit">Add Person</button>
</form>
```

### Controller — HomeController.cs

```csharp
[HttpPost]
public IActionResult ShowIFormAddedPerson(IFormCollection form)
{
    // 'form' contains ALL submitted form fields — read by name key
    ViewData["Id"]       = form["id"];
    ViewData["Name"]     = form["name"];
    ViewData["MobileNo"] = form["mobileNo"];
    ViewData["Address"]  = form["address"];

    return View();
}
```

### Result View — ShowIFormAddedPerson.cshtml

```html
<div>
    <h2>Data Received from the View</h2>
    <p>Id:       @ViewData["Id"]</p>
    <p>Name:     @ViewData["Name"]</p>
    <p>MobileNo: @ViewData["MobileNo"]</p>
    <p>Address:  @ViewData["Address"]</p>
</div>
```

### Request Object vs IFormCollection — What's the Difference?

```
Request.Form["id"]              vs        IFormCollection form → form["id"]
──────────────────────────────            ──────────────────────────────────
Access the global Request object          Receive the form as a parameter
Works anywhere in the controller          Cleaner — dependency is explicit
Fine for quick access                     Slightly more testable & readable
Both do the same thing underneath         Both require [HttpPost]
```

### Key Points

```
✔ Cleaner than Request.Form — form data injected as a parameter
✔ Works like a dictionary — form["fieldName"]
✔ Can loop through all submitted fields if needed
✔ Good when you don't have a model defined yet
✗ Still returns strings — need to convert types manually
✗ Still requires [HttpPost]
✗ Not as clean as Model Binding for structured data
```

## 5. Method 4 — Strongly Typed Model Binding

> **Best method.** You define a Model class with properties that match the form field names. ASP.NET automatically maps the entire form to the model object — no manual reading needed. This is called **Model Binding**.

### How It Works

```
Model class:
  public class Person {
      public int    Id       { get; set; }   ← matches name="id"
      public string Name     { get; set; }   ← matches name="name"
      public long   MobileNo { get; set; }   ← matches name="mobileNo"
      public string Address  { get; set; }   ← matches name="address"
  }

View:
  <input name="id" />         ─┐
  <input name="name" />        ├── ASP.NET maps these to Person automatically
  <input name="mobileNo" />    │
  <input name="address" />    ─┘

Controller:
  public IActionResult ShowModelAddedPerson(Person person)
                                    ↑
                      Fully filled Person object — no manual reading
```

### The Model — Person.cs

```csharp
// Models/Person.cs
namespace TestViewTypes.Models
{
    public class Person
    {
        public int    Id       { get; set; }
        public string Name     { get; set; }
        public long   MobileNo { get; set; }
        public string Address  { get; set; }
    }
}
```

### View — AddModelPerson.cshtml (in Person folder)

```html
<h1>Add Person Form (Model Binding)</h1>

<form name="fr" action="/Person/ShowModelAddedPerson" method="post">
    <div>
        <label>Id</label>
        <input type="number" name="id" />     <!-- must match Person.Id -->
    </div>
    <div>
        <label>Name</label>
        <input type="text" name="name" />     <!-- must match Person.Name -->
    </div>
    <div>
        <label>Mobile No</label>
        <input type="number" name="mobileNo" />
    </div>
    <div>
        <label>Address</label>
        <input type="text" name="address" />
    </div>
    <button type="submit">Add Person</button>
</form>
```

### Controller — PersonController.cs

```csharp
public class PersonController : Controller
{
    public IActionResult AddModelPerson()
    {
        return View();   // Just show the empty form
    }

    [HttpPost]
    public IActionResult ShowModelAddedPerson(Person person)
    //                                         ↑
    //                    ASP.NET auto-fills this entire object from form data
    {
        // 'person' is already fully populated — no manual reading!
        TempData["Id"]       = person.Id;
        TempData["Name"]     = person.Name;
        TempData["MobileNo"] = person.MobileNo;
        TempData["Address"]  = person.Address;

        return View();
    }
}
```

### Result View — ShowModelAddedPerson.cshtml

```html
<div>
    <h2>Data Received from the View</h2>
    <p>Id:       @TempData["Id"]</p>
    <p>Name:     @TempData["Name"]</p>
    <p>MobileNo: @TempData["MobileNo"]</p>
    <p>Address:  @TempData["Address"]</p>
</div>
```

### Why Model Binding is the Best

```
Method 1 (Parameters):          ShowAddedPerson(int id, string name, long mobileNo, string address)
                                 ↑ Gets long and messy as fields grow

Method 2 (Request.Form):        ViewBag.Id = Request.Form["id"];  — 4 repetitive lines

Method 3 (IFormCollection):     ViewData["Id"] = form["id"];      — still repetitive

Method 4 (Model Binding):       ShowModelAddedPerson(Person person)
                                 ↑ ONE parameter, fully populated, typed correctly
                                 ↑ Clean, scalable, no manual reading needed
```

### Key Points

```
✔ Cleanest, most professional approach
✔ ASP.NET maps form fields to model properties automatically
✔ Types are correct (int stays int, long stays long) — no casting
✔ Scales well — add 10 more fields, no change in controller
✔ Works with [Required] and other validation attributes
✔ Matches how real-world MVC projects are built
✗ Requires a Model class to be defined
✗ Input name="" must match property names (case-insensitive)
```

## 6. Side-by-Side Comparison

### All 4 Methods — Same Task, Different Approaches

```
TASK: Read Id, Name, MobileNo, Address from a submitted form
─────────────────────────────────────────────────────────────────────────────────────

METHOD 1 — Action Parameters
  View:        <input name="id" />  <input name="name" />  ... (GET form)
  Controller:  public IActionResult Show(int id, string name, long mobileNo, string address)
  Reading:     Values injected directly as parameters

METHOD 2 — Request Object
  View:        <form method="post">  same inputs
  Controller:  [HttpPost] public IActionResult Show()
               ViewBag.Id   = Request.Form["id"];
               ViewBag.Name = Request.Form["name"];  ...

METHOD 3 — IFormCollection
  View:        <form method="post">  same inputs
  Controller:  [HttpPost] public IActionResult Show(IFormCollection form)
               ViewData["Id"]   = form["id"];
               ViewData["Name"] = form["name"];  ...

METHOD 4 — Model Binding
  View:        <form method="post">  same inputs (name must match properties)
  Controller:  [HttpPost] public IActionResult Show(Person person)
               // person.Id, person.Name already set — no manual reading!
```

### Full Comparison Table

```
┌──────────────────────┬──────────────────┬──────────────────┬──────────────────┬──────────────────┐
│                      │  Parameters      │  Request.Form    │  IFormCollection │  Model Binding   │
├──────────────────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────┤
│ Needs [HttpPost]?    │ ❌ No (GET ok)   │ ✅ Yes           │ ✅ Yes           │ ✅ Yes           │
│ Manual reading?      │ ❌ Auto          │ ✅ Yes, each one │ ✅ Yes, each one │ ❌ Auto          │
│ Type conversion?     │ ❌ Auto          │ ✅ Manual cast   │ ✅ Manual cast   │ ❌ Auto          │
│ Needs a Model class? │ ❌ No            │ ❌ No            │ ❌ No            │ ✅ Yes           │
│ Scales with fields?  │ ❌ Gets messy   │ ❌ Repetitive    │ ⚠️ Repetitive   │ ✅ Very well     │
│ Recommended?         │ ⚠️ Simple cases │ ❌ Not really    │ ⚠️ Sometimes    │ ✅ Always        │
└──────────────────────┴──────────────────┴──────────────────┴──────────────────┴──────────────────┘
```
