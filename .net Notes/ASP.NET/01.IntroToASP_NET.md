## 1. What is MVC?

MVC stands for **Model → View → Controller**.
It is a way to **organise your code** so each part has one job.

```
┌──────────────────────────────────────────────────┐
│                   BROWSER                        │
│         User opens a URL / clicks a button       │
└─────────────────────┬────────────────────────────┘
                      │  HTTP Request
                      ▼
┌──────────────────────────────────────────────────┐
│                  CONTROLLER                      │
│  • Receives the request                          │
│  • Asks BAL/DAL for data                         │
│  • Sends data to the View                        │
└───────────┬──────────────────────┬───────────────┘
            │                      │
            ▼                      ▼
┌───────────────────┐   ┌──────────────────────────┐
│      MODEL        │   │          VIEW            │
│  • C# class       │   │  • .cshtml file          │
│  • Holds data     │◄──│  • HTML + Razor          │
│  • Validation     │   │  • Shows data to user    │
└───────────────────┘   └──────────────────────────┘
```

| Part                 | Job                           | Example File             |
| -------------------- | ----------------------------- | ------------------------ |
| **Model**      | Holds data & validation rules | `Student.cs`           |
| **View**       | Shows HTML to the user        | `Index.cshtml`         |
| **Controller** | Connects Model and View       | `StudentController.cs` |

## 3. Program.cs

> This is the **starting point** of your app.It does two things:
>
> - **Register services** — tell the app what to use
> - **Set up middleware** — control how requests are handled

```csharp
// Program.cs

var builder = WebApplication.CreateBuilder(args);

// ───────────────────────────────────
// STEP 1: Register Services
// ───────────────────────────────────

// Tell the app we are using MVC (Controllers + Views)
builder.Services.AddControllersWithViews();

var app = builder.Build();

// ───────────────────────────────────
// STEP 2: Set Up Middleware Pipeline
// ───────────────────────────────────

// Show a friendly error page when something goes wrong (in production)
if (!app.Environment.IsDevelopment())
{
    app.UseExceptionHandler("/Home/Error");
}

// Redirect HTTP to HTTPS
app.UseHttpsRedirection();

// Allow the app to serve CSS, JS, images from wwwroot
app.UseStaticFiles();

// Enable routing (so URLs map to controllers)
app.UseRouting();

// Default URL pattern: /ControllerName/ActionName/id
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");

app.Run();
```

### What is Middleware?

Middleware is like a series of checkpoints every request goes through:

```
Request comes in
      │
      ▼
 UseHttpsRedirection   → Force HTTPS
      │
      ▼
 UseStaticFiles        → If URL is a file (CSS/JS) → serve it, stop here
      │
      ▼
 UseRouting            → Match URL to a Controller + Action
      │
      ▼
 MapControllerRoute    → Run the Controller Action
      │
      ▼
Response sent to browser
```

## 4. appsettings.json

> Stores **configuration** your app needs — like the database connection string.
> Never hardcode passwords or connection strings directly in your C# code. Put them here.

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=YOUR_SERVER;Database=SchoolDB;Trusted_Connection=True;"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
```

### How to Read Connection String in C#

```csharp
// In your DAL class — inject IConfiguration
private readonly string _connectionString;

public StudentDAL(IConfiguration config)
{
    // Reads from appsettings.json → ConnectionStrings → DefaultConnection
    _connectionString = config.GetConnectionString("DefaultConnection")!;
}
```

### appsettings.Development.json

This file **overrides** `appsettings.json` only when you run the app locally on your machine during development.

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.;Database=SchoolDB_Dev;Trusted_Connection=True;"
  }
}
```

## 5. Model

> A Model is just a **C# class** that represents your data.
> It matches the shape of your database table.

```csharp
// Models/Student.cs
using System.ComponentModel.DataAnnotations;

namespace MyMvcApp.Models
{
    public class Student
    {
        public int Id { get; set; }

        [Required(ErrorMessage = "Name is required")]
        [Display(Name = "Full Name")]
        public string Name { get; set; } = string.Empty;

        [Required]
        [Range(5, 100, ErrorMessage = "Age must be between 5 and 100")]
        public int Age { get; set; }

        [Required]
        [EmailAddress(ErrorMessage = "Enter a valid email")]
        public string Email { get; set; } = string.Empty;

        [Display(Name = "Enrolled On")]
        public DateTime EnrolledDate { get; set; } = DateTime.Now;
    }
}
```

### Common Data Annotations (Validation Rules)

| Annotation                | What It Does                 |
| ------------------------- | ---------------------------- |
| `[Required]`            | Field cannot be empty        |
| `[Display(Name="...")]` | Label shown in forms         |
| `[Range(1, 100)]`       | Number must be in this range |
| `[StringLength(50)]`    | Max characters allowed       |
| `[EmailAddress]`        | Must look like an email      |
| `[MinLength(3)]`        | Minimum characters           |

## 6. Controller

> The Controller **handles requests**.
> When someone visits a URL, the right controller action runs.

```csharp
// Controllers/StudentController.cs
using Microsoft.AspNetCore.Mvc;
using MyMvcApp.BAL;
using MyMvcApp.Models;

namespace MyMvcApp.Controllers
{
    public class StudentController : Controller
    {
        // We use BAL — controller should NOT talk to DB directly
        private readonly StudentBAL _studentBAL;

        public StudentController(StudentBAL studentBAL)
        {
            _studentBAL = studentBAL;
        }

        // ─────────────────────────────
        // GET: /Student  OR  /Student/Index
        // Shows list of all students
        // ─────────────────────────────
        public IActionResult Index()
        {
            var students = _studentBAL.GetAllStudents();
            return View(students);   // Sends list to Index.cshtml
        }

        // ─────────────────────────────
        // GET: /Student/Details/3
        // Show one student's details
        // ─────────────────────────────
        public IActionResult Details(int id)
        {
            var student = _studentBAL.GetStudentById(id);

            if (student == null)
                return NotFound();   // Returns a 404 page

            return View(student);
        }

        // ─────────────────────────────
        // GET: /Student/Create
        // Show an empty form
        // ─────────────────────────────
        public IActionResult Create()
        {
            return View();
        }

        // ─────────────────────────────
        // POST: /Student/Create
        // User submits the form
        // ─────────────────────────────
        [HttpPost]
        [ValidateAntiForgeryToken]   // Security — prevents CSRF attacks
        public IActionResult Create(Student student)
        {
            if (ModelState.IsValid)  // Check if all validation rules pass
            {
                _studentBAL.AddStudent(student);
                TempData["Message"] = "Student added successfully!";
                return RedirectToAction("Index");  // Go back to list
            }

            return View(student);  // Show form again with errors
        }

        // ─────────────────────────────
        // GET: /Student/Edit/3
        // Show form filled with student data
        // ─────────────────────────────
        public IActionResult Edit(int id)
        {
            var student = _studentBAL.GetStudentById(id);
            if (student == null) return NotFound();
            return View(student);
        }

        // ─────────────────────────────
        // POST: /Student/Edit/3
        // User submits edited form
        // ─────────────────────────────
        [HttpPost]
        [ValidateAntiForgeryToken]
        public IActionResult Edit(Student student)
        {
            if (ModelState.IsValid)
            {
                _studentBAL.UpdateStudent(student);
                TempData["Message"] = "Student updated!";
                return RedirectToAction("Index");
            }

            return View(student);
        }

        // ─────────────────────────────
        // GET: /Student/Delete/3
        // Show confirmation page
        // ─────────────────────────────
        public IActionResult Delete(int id)
        {
            var student = _studentBAL.GetStudentById(id);
            if (student == null) return NotFound();
            return View(student);
        }

        // ─────────────────────────────
        // POST: /Student/Delete/3
        // User confirms delete
        // ─────────────────────────────
        [HttpPost, ActionName("Delete")]
        [ValidateAntiForgeryToken]
        public IActionResult DeleteConfirmed(int id)
        {
            _studentBAL.DeleteStudent(id);
            TempData["Message"] = "Student deleted!";
            return RedirectToAction("Index");
        }
    }
}
```

### Common Return Types from a Controller

| What You Write                       | What Happens                         |
| ------------------------------------ | ------------------------------------ |
| `return View()`                    | Render the matching `.cshtml` view |
| `return View(model)`               | Render view and pass data to it      |
| `return RedirectToAction("Index")` | Redirect browser to another action   |
| `return NotFound()`                | Return a 404 page                    |
| `return Content("Hello")`          | Return plain text                    |

### URL to Controller Mapping

```
URL: /Student/Edit/5
       │        │   │
       │        │   └── id = 5   (passed as parameter)
       │        └────── Action = Edit
       └─────────────── Controller = Student
```

## 7. Views & Razor Basics

> Views are `.cshtml` files — HTML with some **C# mixed in** using the `@` symbol.
> The C# part is called **Razor syntax**.

### Index View (List Page)

```html
<!-- Views/Student/Index.cshtml -->
@model IEnumerable<MyMvcApp.Models.Student>

@{
    ViewData["Title"] = "Students";
}

<h2>All Students</h2>

@* Show success/delete message if it exists *@
@if (TempData["Message"] != null)
{
    <div class="alert alert-success">@TempData["Message"]</div>
}

<a asp-action="Create" class="btn btn-primary mb-3">+ Add New Student</a>

<table class="table table-bordered">
    <thead class="table-dark">
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>Email</th>
            <th>Actions</th>
        </tr>
    </thead>
    <tbody>
        @foreach (var student in Model)
        {
            <tr>
                <td>@student.Name</td>
                <td>@student.Age</td>
                <td>@student.Email</td>
                <td>
                    <a asp-action="Details" asp-route-id="@student.Id" class="btn btn-sm btn-info">Details</a>
                    <a asp-action="Edit"    asp-route-id="@student.Id" class="btn btn-sm btn-warning">Edit</a>
                    <a asp-action="Delete"  asp-route-id="@student.Id" class="btn btn-sm btn-danger">Delete</a>
                </td>
            </tr>
        }
    </tbody>
</table>
```

### Create View (Form Page)

```html
<!-- Views/Student/Create.cshtml -->
@model MyMvcApp.Models.Student

@{
    ViewData["Title"] = "Add Student";
}

<h2>Add New Student</h2>

<form asp-action="Create" method="post">
    @Html.AntiForgeryToken()

    @* Show all validation errors in one place *@
    <div asp-validation-summary="All" class="text-danger mb-3"></div>

    <div class="mb-3">
        <label asp-for="Name" class="form-label"></label>
        <input asp-for="Name" class="form-control" />
        <span asp-validation-for="Name" class="text-danger"></span>
    </div>

    <div class="mb-3">
        <label asp-for="Age" class="form-label"></label>
        <input asp-for="Age" class="form-control" />
        <span asp-validation-for="Age" class="text-danger"></span>
    </div>

    <div class="mb-3">
        <label asp-for="Email" class="form-label"></label>
        <input asp-for="Email" class="form-control" />
        <span asp-validation-for="Email" class="text-danger"></span>
    </div>

    <button type="submit" class="btn btn-success">Save</button>
    <a asp-action="Index" class="btn btn-secondary">Cancel</a>
</form>
```

### Razor Syntax — Quick Reference

```cshtml
@* This is a comment — not sent to browser *@

@Model.Name                      ← Display a property from Model
@{ var x = 10; }                 ← C# code block
@x                               ← Display a variable

@if (Model.Age > 18)
{
    <p>Adult</p>
}

@foreach (var item in Model)
{
    <p>@item.Name</p>
}
```

### Tag Helpers (`asp-*`)

Tag helpers are special HTML attributes that generate correct HTML for you:

```html
<!-- Link to another page -->
<a asp-controller="Student" asp-action="Index">Back to list</a>

<!-- Link with an ID -->
<a asp-action="Edit" asp-route-id="@student.Id">Edit</a>

<!-- Form that posts to an action -->
<form asp-action="Create" method="post">

<!-- Input linked to a model property (sets name, id, type automatically) -->
<input asp-for="Name" class="form-control" />

<!-- Label linked to a model property (uses [Display(Name=...)] if set) -->
<label asp-for="Name"></label>

<!-- Shows validation error for a specific field -->
<span asp-validation-for="Name" class="text-danger"></span>
```

## 8. Shared — `_Layout.cshtml`

> Think of `_Layout.cshtml` as a **template / master page**.
> It defines the outer shell — header, navbar, footer — that wraps **every page** in your app.

```html
<!-- Views/Shared/_Layout.cshtml -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>@ViewData["Title"] — School App</title>

    <!-- Bootstrap CSS from wwwroot -->
    <link rel="stylesheet" href="~/lib/bootstrap/dist/css/bootstrap.min.css" />
    <link rel="stylesheet" href="~/css/site.css" />
</head>
<body>

    <!-- Navigation Bar -->
    <nav class="navbar navbar-expand-lg navbar-dark bg-primary">
        <div class="container">
            <a class="navbar-brand" asp-controller="Home" asp-action="Index">🏫 School App</a>
            <div class="navbar-nav">
                <a class="nav-link" asp-controller="Home" asp-action="Index">Home</a>
                <a class="nav-link" asp-controller="Student" asp-action="Index">Students</a>
            </div>
        </div>
    </nav>

    <!-- Main Content Area — each page fills this -->
    <div class="container mt-4">

        @RenderBody()   ← THIS IS WHERE EACH VIEW'S HTML GOES

    </div>

    <!-- Footer -->
    <footer class="text-center mt-5 py-3 bg-light">
        <p>© @DateTime.Now.Year School App</p>
    </footer>

    <!-- JS files -->
    <script src="~/lib/jquery/dist/jquery.min.js"></script>
    <script src="~/lib/bootstrap/dist/js/bootstrap.bundle.min.js"></script>
    <script src="~/js/site.js"></script>

    <!-- Each page can add its own scripts here -->
    @await RenderSectionAsync("Scripts", required: false)

</body>
</html>
```

### How Layout Wraps Your View

```
_Layout.cshtml
┌────────────────────────────────────────────┐
│  <html>                                    │
│    <head> ... CSS links ... </head>        │
│    <body>                                  │
│      <nav> Navbar here </nav>              │
│                                            │
│      ┌──────────────────────────────────┐  │
│      │         @RenderBody()            │  │
│      │   ← Your view's HTML goes here  │  │
│      │   e.g., Index.cshtml content    │  │
│      └──────────────────────────────────┘  │
│                                            │
│      <footer> Footer here </footer>        │
│      <script> JS links </script>           │
│    </body>                                 │
│  </html>                                   │
└────────────────────────────────────────────┘
```

### Adding Page-Specific Scripts

In your child view, use `@section Scripts` to add JS that only loads on that page:

```html
@* At the bottom of Create.cshtml *@
@section Scripts {
    <script src="~/lib/jquery-validation/dist/jquery.validate.min.js"></script>
    <script>
        console.log("This only runs on Create page");
    </script>
}
```

## 9. `_ViewStart.cshtml` & `_ViewImports.cshtml`

### _ViewStart.cshtml

> Runs **before every view** automatically.
> Its only job is to set the default layout, so you don't have to write it in every `.cshtml` file.

```cshtml
@* Views/_ViewStart.cshtml *@
@{
    Layout = "_Layout";
}
```

If one specific page should have **no layout** (like a login page), override it in that view:

```cshtml
@* In that specific view *@
@{ Layout = null; }
```

### _ViewImports.cshtml

> Applies `@using` namespaces and Tag Helpers to **all views at once**.
> Without this, you'd need to add `@using MyMvcApp.Models` at the top of every single view.

```cshtml
@* Views/_ViewImports.cshtml *@

@using MyMvcApp
@using MyMvcApp.Models

@* This line enables all the asp-* tag helpers *@
@addTagHelper *, Microsoft.AspNetCore.Mvc.TagHelpers
```

## 11. DAL — Data Access Layer with ADO.NET

> DAL = **Data Access Layer**.
> This is where **all database code lives**.
> We use **ADO.NET** — you write SQL queries yourself and run them with C# code.

### What is ADO.NET?

ADO.NET is a set of C# classes that let you talk to a database by writing **raw SQL**:

```
Your C# Code
     │
     ▼
SqlConnection   → Opens connection to the database
     │
     ▼
SqlCommand      → Holds your SQL query
     │
     ▼
SqlDataReader   → Reads rows returned by SELECT
     │
     ▼
Your Model / List
```

### StudentDAL.cs — Full Example

```csharp
// DAL/StudentDAL.cs
using System.Data;
using Microsoft.Data.SqlClient;          // ADO.NET namespace
using MyMvcApp.Models;

namespace MyMvcApp.DAL
{
    public class StudentDAL
    {
        private readonly string _connectionString;

        // IConfiguration reads from appsettings.json
        public StudentDAL(IConfiguration config)
        {
            _connectionString = config.GetConnectionString("DefaultConnection")!;
        }

        // ─────────────────────────────────────────
        // GET ALL STUDENTS
        // ─────────────────────────────────────────
        public List<Student> GetAllStudents()
        {
            var students = new List<Student>();

            // 'using' automatically closes the connection when done
            using (var con = new SqlConnection(_connectionString))
            {
                var cmd = new SqlCommand("SELECT * FROM Students", con);
                con.Open();

                using (var reader = cmd.ExecuteReader())
                {
                    while (reader.Read())   // Loop through each row
                    {
                        students.Add(new Student
                        {
                            Id    = Convert.ToInt32(reader["Id"]),
                            Name  = reader["Name"].ToString()!,
                            Age   = Convert.ToInt32(reader["Age"]),
                            Email = reader["Email"].ToString()!
                        });
                    }
                }
            }

            return students;
        }

        // ─────────────────────────────────────────
        // GET ONE STUDENT BY ID
        // ─────────────────────────────────────────
        public Student? GetStudentById(int id)
        {
            Student? student = null;

            using (var con = new SqlConnection(_connectionString))
            {
                // Use parameters — NEVER put id directly in the SQL string
                // Writing "WHERE Id = " + id  is dangerous (SQL Injection)
                var cmd = new SqlCommand("SELECT * FROM Students WHERE Id = @Id", con);
                cmd.Parameters.AddWithValue("@Id", id);   // Safe ✅
                con.Open();

                using (var reader = cmd.ExecuteReader())
                {
                    if (reader.Read())
                    {
                        student = new Student
                        {
                            Id    = Convert.ToInt32(reader["Id"]),
                            Name  = reader["Name"].ToString()!,
                            Age   = Convert.ToInt32(reader["Age"]),
                            Email = reader["Email"].ToString()!
                        };
                    }
                }
            }

            return student;
        }

        // ─────────────────────────────────────────
        // ADD A NEW STUDENT (INSERT)
        // ─────────────────────────────────────────
        public void AddStudent(Student student)
        {
            using (var con = new SqlConnection(_connectionString))
            {
                var cmd = new SqlCommand(
                    "INSERT INTO Students (Name, Age, Email) VALUES (@Name, @Age, @Email)", con);

                cmd.Parameters.AddWithValue("@Name",  student.Name);
                cmd.Parameters.AddWithValue("@Age",   student.Age);
                cmd.Parameters.AddWithValue("@Email", student.Email);

                con.Open();
                cmd.ExecuteNonQuery();   // Use this for INSERT, UPDATE, DELETE
            }
        }

        // ─────────────────────────────────────────
        // UPDATE AN EXISTING STUDENT
        // ─────────────────────────────────────────
        public void UpdateStudent(Student student)
        {
            using (var con = new SqlConnection(_connectionString))
            {
                var cmd = new SqlCommand(
                    "UPDATE Students SET Name = @Name, Age = @Age, Email = @Email WHERE Id = @Id", con);

                cmd.Parameters.AddWithValue("@Id",    student.Id);
                cmd.Parameters.AddWithValue("@Name",  student.Name);
                cmd.Parameters.AddWithValue("@Age",   student.Age);
                cmd.Parameters.AddWithValue("@Email", student.Email);

                con.Open();
                cmd.ExecuteNonQuery();
            }
        }

        // ─────────────────────────────────────────
        // DELETE A STUDENT
        // ─────────────────────────────────────────
        public void DeleteStudent(int id)
        {
            using (var con = new SqlConnection(_connectionString))
            {
                var cmd = new SqlCommand("DELETE FROM Students WHERE Id = @Id", con);
                cmd.Parameters.AddWithValue("@Id", id);

                con.Open();
                cmd.ExecuteNonQuery();
            }
        }
    }
}
```

### ADO.NET — Key Classes Explained

| Class                         | What it Does                                            |
| ----------------------------- | ------------------------------------------------------- |
| `SqlConnection`             | Opens/closes connection to SQL Server                   |
| `SqlCommand`                | Holds your SQL query or stored procedure                |
| `SqlDataReader`             | Reads rows one by one from a SELECT result              |
| `cmd.ExecuteReader()`       | Use for**SELECT** — returns rows                 |
| `cmd.ExecuteNonQuery()`     | Use for**INSERT / UPDATE / DELETE**               |
| `cmd.ExecuteScalar()`       | Use when query returns**one value** (e.g., COUNT) |
| `Parameters.AddWithValue()` | Safely add values to SQL (prevents SQL Injection)       |

### ⚠️ SQL Injection Warning

```csharp
// ❌ WRONG — DANGEROUS — Never do this
var cmd = new SqlCommand("SELECT * FROM Students WHERE Id = " + id, con);

// ✅ CORRECT — Always use parameters
var cmd = new SqlCommand("SELECT * FROM Students WHERE Id = @Id", con);
cmd.Parameters.AddWithValue("@Id", id);
```

## 12. BAL — Business Access Layer

> BAL = **Business Access Layer** (also called Service Layer).
> It sits between the Controller and DAL.
> It holds **business rules** — things like "a student's age must be over 5" or "email must be unique".

The controller should be **thin** (just handle requests).
The DAL should **only** talk to the database.
Business logic goes in the BAL.

```csharp
// BAL/StudentBAL.cs
using MyMvcApp.DAL;
using MyMvcApp.Models;

namespace MyMvcApp.BAL
{
    public class StudentBAL
    {
        private readonly StudentDAL _studentDAL;

        public StudentBAL(StudentDAL studentDAL)
        {
            _studentDAL = studentDAL;
        }

        public List<Student> GetAllStudents()
        {
            return _studentDAL.GetAllStudents();
        }

        public Student? GetStudentById(int id)
        {
            return _studentDAL.GetStudentById(id);
        }

        public void AddStudent(Student student)
        {
            // Business rule: Set enrolled date before saving
            student.EnrolledDate = DateTime.Now;

            _studentDAL.AddStudent(student);
        }

        public void UpdateStudent(Student student)
        {
            _studentDAL.UpdateStudent(student);
        }

        public void DeleteStudent(int id)
        {
            _studentDAL.DeleteStudent(id);
        }
    }
}
```

### Register DAL and BAL in Program.cs

Because we use Dependency Injection, we must register our classes:

```csharp
// Add these lines to Program.cs — before var app = builder.Build();

builder.Services.AddScoped<StudentDAL>();
builder.Services.AddScoped<StudentBAL>();
```

`AddScoped` means: create a new instance of this class for every HTTP request.

## 13. ADO.NET vs EF Core — What's the Difference?

> **Yes — ADO.NET can absolutely replace EF Core.**
> They both connect to a database. The difference is *how much you write yourself*.

```
┌─────────────────────────────────────────────────────────────────┐
│                        DATABASE                                 │
└────────────────────────────┬────────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │                             │
     ┌────────▼──────────┐       ┌──────────▼──────────┐
     │     ADO.NET       │       │      EF Core         │
     │                   │       │                      │
     │  You write SQL    │       │  EF writes SQL for   │
     │  yourself         │       │  you automatically   │
     │                   │       │                      │
     │  More control     │       │  Less code           │
     │  More typing      │       │  Easier to start     │
     └───────────────────┘       └──────────────────────┘
```

| Feature              | ADO.NET                  | EF Core                          |
| -------------------- | ------------------------ | -------------------------------- |
| Write SQL?           | Yes — you write it      | No — EF generates it            |
| Speed                | Slightly faster          | Slightly slower                  |
| Learning curve       | Easier to understand SQL | Easier to start, harder to debug |
| Stored procedures    | Easy                     | Possible but awkward             |
| Beginner friendly?   | Good if you know SQL     | Good if you don't know SQL       |
| Control over queries | Full control             | Limited                          |

**Simple rule:**

- Know SQL? → ADO.NET is fine and gives you full control.
- Don't know SQL yet? → EF Core is quicker to get started.
