What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organises code into objects — self-contained units that combine data (fields) and behaviour (methods).

OOP is built on four core pillars:

🔒
Encapsulation
Hiding internal data and exposing only what's needed through controlled access.
🧬
Inheritance
A class acquires properties and behaviour of another class, enabling code reuse.
🎭
Polymorphism
One interface, many implementations — same method behaves differently for different types.
🎨
Abstraction
Hiding implementation details and showing only the essential features.

Class & Object

A Class is a blueprint or template. An Object is an instance of a class — the actual thing created from the blueprint.

C#
// Class — blueprint
class Car
{
    // Fields (data)
    public string brand;
    public int    year;

    // Method (behaviour)
    public void Start()
    {
        Console.WriteLine(brand + " is starting...");
    }
}

// Object — instance of the class
Car myCar = new Car();
myCar.brand = "Toyota";
myCar.year  = 2022;
myCar.Start();   // Output: Toyota is starting...

Constructors

A constructor is a special method that runs automatically when an object is created. It initialises the object's state.

Types of Constructors

C#
class Student
{
    public string name;
    public int    age;

    // 1. Default constructor (no parameters)
    public Student()
    {
        name = "Unknown";
        age  = 0;
    }

    // 2. Parameterised constructor
    public Student(string name, int age)
    {
        this.name = name;
        this.age  = age;
    }

    // 3. Copy constructor
    public Student(Student other)
    {
        this.name = other.name;
        this.age  = other.age;
    }
}

Student s1 = new Student();                    // default
Student s2 = new Student("Sumit", 22);          // parameterised
Student s3 = new Student(s2);                   // copy

C# vs Java: C# uses this.field = value just like Java. If no constructor is written, C# provides a default one — same as Java. C# also supports constructor chaining using : this().

Encapsulation

Encapsulation means wrapping data (fields) and methods inside a class, and restricting direct access from outside using access modifiers.

Access Modifiers in C#

ModifierAccessible From
publicAnywhere
privateWithin the same class only
protectedWithin the class and its derived (child) classes
internalWithin the same assembly (project)
protected internalSame assembly OR derived classes
Example
class BankAccount
{
    private double balance;    // hidden from outside

    public void Deposit(double amount)
    {
        if (amount > 0) balance += amount;
    }

    public double GetBalance()
    {
        return balance;       // controlled read access
    }
}

Properties — C# Smart Getters & Setters

C# has a cleaner way to expose private fields: Properties. Instead of separate getXxx() / setXxx() methods like Java, C# uses built-in get / set accessors.

Full Property
class Person
{
    private string _name;    // backing field

    public string Name       // property
    {
        get { return _name; }
        set
        {
            if (!string.IsNullOrEmpty(value))
                _name = value;
        }
    }
}
Auto-Property (shorthand)
class Person
{
    public string  Name  { get; set; }   // read & write
    public int     Age   { get; set; }
    public string  Id    { get; }         // read-only
}

// Usage (same as Java getters/setters, but cleaner)
Person p = new Person();
p.Name = "Sumit";
Console.WriteLine(p.Name);

Inheritance

Inheritance allows a class to acquire the fields and methods of another class. The class being inherited from is the base (parent) class; the class that inherits is the derived (child) class.

C# uses : for inheritance (Java uses extends).

Example
class Animal                      // base class
{
    public string Name;
    public void Eat() => Console.WriteLine(Name + " is eating");
}

class Dog : Animal               // derived class (C# syntax)
{
    public void Bark() => Console.WriteLine(Name + " says: Woof!");
}

Dog d = new Dog();
d.Name = "Bruno";
d.Eat();    // inherited from Animal
d.Bark();   // own method

Calling Parent Constructor — base()

base() — like super() in Java
class Vehicle
{
    public string Brand;
    public Vehicle(string brand) { Brand = brand; }
}

class Car : Vehicle
{
    public int Doors;
    public Car(string brand, int doors) : base(brand)
    {
        Doors = doors;
    }
}

Car c = new Car("Honda", 4);
Console.WriteLine(c.Brand + " with " + c.Doors + " doors");

Important: C# supports single inheritance only (a class can inherit from only one class). To achieve multiple inheritance-like behaviour, use interfaces. Use sealed keyword to prevent a class from being inherited.

Polymorphism

Polymorphism means "many forms" — the same method name behaves differently depending on the object. C# has two types:

1. Compile-time Polymorphism — Method Overloading

Same method name, different parameter lists, resolved at compile time.

Method Overloading
class Calculator
{
    public int    Add(int a, int b)               => a + b;
    public double Add(double a, double b)         => a + b;
    public int    Add(int a, int b, int c)       => a + b + c;
}

Calculator calc = new Calculator();
Console.WriteLine(calc.Add(2, 3));         // 5
Console.WriteLine(calc.Add(1.5, 2.5));     // 4.0
Console.WriteLine(calc.Add(1, 2, 3));      // 6

2. Runtime Polymorphism — Method Overriding

Child class provides its own implementation of a method defined in the parent class. Resolved at runtime. C# requires virtual in the parent and override in the child (Java uses @Override annotation).

virtual + override
class Shape
{
    public virtual void Draw()     // virtual = can be overridden
        => Console.WriteLine("Drawing a Shape");
}

class Circle : Shape
{
    public override void Draw()   // override = replacing parent implementation
        => Console.WriteLine("Drawing a Circle");
}

class Rectangle : Shape
{
    public override void Draw()
        => Console.WriteLine("Drawing a Rectangle");
}

// Runtime polymorphism via base class reference
Shape s1 = new Circle();
Shape s2 = new Rectangle();
s1.Draw();   // Drawing a Circle
s2.Draw();   // Drawing a Rectangle

C# vs Java difference: In Java, all non-static methods are overridable by default. In C#, you must explicitly mark a method as virtual to allow overriding. Use sealed override to prevent further overriding in subclasses.

Abstraction

Abstraction means hiding implementation details and showing only the essential features. In C#, abstraction is achieved using abstract classes and interfaces.

Abstract Class

An abstract class cannot be instantiated directly. It can contain both abstract methods (no body) and concrete methods (with body).

Abstract Class Example
abstract class Animal
{
    public string Name;

    // Abstract method — no body, MUST be implemented by child class
    public abstract void MakeSound();

    // Concrete method — has body, can be inherited as-is
    public void Sleep() => Console.WriteLine(Name + " is sleeping.");
}

class Dog : Animal
{
    public override void MakeSound()    // must implement abstract method
        => Console.WriteLine("Woof!");
}

class Cat : Animal
{
    public override void MakeSound()
        => Console.WriteLine("Meow!");
}

// Animal a = new Animal(); ❌ Cannot instantiate abstract class
Animal dog = new Dog() { Name = "Bruno" };
dog.MakeSound();   // Woof!
dog.Sleep();       // Bruno is sleeping.

Interfaces

An interface is a contract. It defines what a class must do but not how to do it. A class that implements an interface must implement all its members.

C# uses : to implement interfaces (Java uses implements). A class can implement multiple interfaces.

Interface Example
interface IFlyable
{
    void Fly();                  // no body — just signature
    string Description { get; } // interface can have properties too
}

interface ISwimmable
{
    void Swim();
}

// A class implementing multiple interfaces
class Duck : IFlyable, ISwimmable
{
    public string Description => "I am a Duck";

    public void Fly()  => Console.WriteLine("Duck flying");
    public void Swim() => Console.WriteLine("Duck swimming");
}

Duck d = new Duck();
d.Fly();
d.Swim();

// Interface reference — polymorphism
IFlyable flyer = new Duck();
flyer.Fly();

static Keyword

A static member belongs to the class itself, not to any instance. There is only one copy, shared by all objects.

Static Field, Method, Class
class Counter
{
    public static int Count = 0;    // shared across all instances

    public Counter() { Count++; }

    public static void Reset() { Count = 0; }
}

new Counter(); new Counter(); new Counter();
Console.WriteLine(Counter.Count);   // 3  (accessed via class name)
Counter.Reset();
Console.WriteLine(Counter.Count);   // 0
